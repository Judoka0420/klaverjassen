# Klaverjassen — mobile app

A Capacitor wrapper around the Klaverjassen web game. Three modes:

- **Solo (offline)** — play vs bots with no network at all.
- **Host a game** — your phone becomes the authoritative game (runs `game-room.js`
  locally) and accepts friends over a direct WebRTC data channel.
- **Join a game** — connect to a friend's hosted table with a 4-letter code.

Remote play uses **PeerJS** (free public matchmaker + Google STUN) for peer-to-peer
connections — there is **no server to run**. Everyone needs internet; the host must
keep the app open. A small fraction of strict/CGNAT networks may fail to connect
(no free TURN relay).

## Layout

```
app/
├─ www/                     ← the web app (Capacitor webDir)
│  ├─ index.html            ← UI + mode picker (Solo / Host / Join)
│  ├─ net.js                ← transport abstraction (inproc / peer-host / peer-client)
│  └─ lib/                  ← generated by scripts/sync-assets.js (git-ignored)
│     ├─ engine.js          ← copied from repo root (single source of truth)
│     ├─ game-room.js       ← copied from repo root
│     └─ peerjs.min.js      ← copied from node_modules
├─ scripts/sync-assets.js   ← copies the above into www/lib
├─ capacitor.config.json
└─ android/                 ← native Gradle project (Capacitor)
```

The rules (`engine.js`) and game flow (`game-room.js`) are the **same files the Node
server uses** — `sync-assets.js` copies them in so the app can never drift from the
server implementation.

## Test in a browser (fastest)

```bash
cd app
npm install
npm run sync-assets
npx serve www          # then open the printed http://localhost:… URL
```

Solo works fully offline. Host/Join need internet (PeerJS). Open two browser
tabs/windows to try host + join against yourself.

## Get an installable Android APK (no local Android SDK needed)

The APK is built in the cloud by GitHub Actions:

1. Push this repository (root = the `klaverjassen` folder) to GitHub.
2. The **Build Android APK** workflow runs on push (or trigger it manually from the
   Actions tab → *Run workflow*).
3. Open the finished run → **Artifacts** → download `klaverjassen-debug-apk`.
4. Copy the `.apk` to an Android phone and open it. Enable *Install unknown apps*
   for your browser/file manager when prompted.

The debug APK is signed with the standard debug key and installs on any device — no
Play Store account required. To publish on the Play Store later, add a release keystore
and a signed `assembleRelease` step.

## Build locally (optional — requires Android Studio + JDK 17)

```bash
cd app
npm install
npm run cap:sync        # sync-assets + cap sync
cd android
./gradlew assembleDebug # APK at app/build/outputs/apk/debug/app-debug.apk
```

## iOS (later)

The same Capacitor project supports iOS (`npx cap add ios`), but building/signing
iOS requires macOS. Without a Mac, use a cloud-Mac CI (e.g. Codemagic) plus an Apple
Developer account when you're ready.

## Follow-ups / polish

- **App icon & splash** — currently the Capacitor default. Add a source icon and run
  `@capacitor/assets` to generate all densities.
- **Reconnect UX** if the host drops (game currently ends).
- **TURN relay** if friends on strict networks can't connect.
